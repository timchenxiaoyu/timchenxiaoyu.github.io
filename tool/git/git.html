# Git

# config

git的配置范围有三种分别是：global、system和local

  • global代表全局
  • system代表每个登录用户，不太常用
  • local代表本地仓库

其中local的优先级高于global，在开发中通常将公司的仓库的配置设置为global

# git config --local user.name 'timechenxiaoyu'
# git config --local user.email '8378xxxx@qq.com'
1
2

获取配置可以通过

# git config --local --list
# git config --global --list
1
2

# init

可以在项目目录下执行"git init"初始化，也可以直接创建

# git init gittest
1

# add

将工作区修改提交到暂存区

# git add 文件或者目录
1

# commit

将暂存区修改合并到本地仓库

# git commit -m"注释"

# checkout

切换分支，如果是新建分支需要"-b"，可以从当前分支拉出新分支如：

# git checkout -b dev
Switched to a new branch 'dev'
1
2

还可以基于某一次commit创建分支：

# git checkout -b fixbug 5dc33d0835818d4beee23fcfa75743b5e034485a
Switched to a new branch 'fixbug'
1
2
